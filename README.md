## 🧭 Methodology: CRISP-DM + Lean

Projects follow CRISP-DM as the primary structure — not because it was taught,
but because it forces the right question first: *what business decision does this support?*

Lean principles eliminate waste at every phase:
- No exhaustive EDA — only what informs the decision
- No over-engineered models — simplest model that solves the problem
- No technical-only output — every project ends with a business-readable conclusion

The analytical chain always runs:

```
Business Problem → Decision to Support → Data → Model → Business Impact
```

> Projects that start at "model" look junior.
> Projects that start at "business problem" look senior.

## 🧭 Metodología: CRISP-DM + Lean

Los proyectos siguen CRISP-DM como estructura principal — no porque fue enseñado, sino porque fuerza la pregunta correcta primero: *¿qué decisión de negocio apoya este análisis?*

Los principios Lean eliminan el desperdicio en cada fase:
- Sin EDA exhaustivo — solo lo que informa la decisión
- Sin modelos sobreingeniados — el modelo más simple que resuelve el problema
- Sin output solo técnico — cada proyecto termina con una conclusión legible para el negocio

La cadena analítica siempre corre:

```
Problema de Negocio → Decisión a Apoyar → Datos → Modelo → Impacto de Negocio
```

> Los proyectos que parten del "modelo" se ven junior.
> Los proyectos que parten del "problema de negocio" se ven senior.
